Campaigns
A campaign is work that spans multiple sessions. State persists to disk and resumes exactly where it left off.
Starting a campaign
/archon build a notifications system — or let /do route there automatically. Archon creates .planning/campaigns/{slug}.md before executing anything.
Phase types
research → plan → build → wire → verify → prune. Each has machine-verifiable end conditions — not marked complete until they pass.
Automatic self-correction
· Direction alignment every 2 phases
· Quality spot-check every phase
· Regression guard after build phases
· Anti-pattern scan on all modified files
Resuming
Run /do continue. Archon reads the Active Context section and picks up at the exact sub-step.
Archon vs Fleet
Archon — sequential phases, one agent per phase. Best for single-domain complex work.

Fleet — parallel agents in isolated worktrees per wave. Best for platform-wide independent tracks.
Circuit breakers
Parks when: 3+ consecutive failures on the same approach · 5+ new type errors in one phase · direction drift detected twice · quality bar fails 3 times.
